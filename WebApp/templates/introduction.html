数据来源
Project 起点

Github代码托管平台

知乎网络问答社区

分析方式
Project 可行性

Github使用提供的API接口https://developer.github.com/v3/, 分析方式为starred数较高的仓库作为数据集, 采集要点是: 开发语言, 简介词条, 和频繁commit的时间(精确到小时);

知乎平台的数据分析, 利用知乎便于进行爬虫的特点, 采用爬取关注, 收藏度较高的作为数据集, 采集要点是: 知乎提供的标 签分类, 回答者的公共信息, 回答者的关注话题 等;

工具知识
Project 核心

数据获取: Srapy, BeautifulSoup, 正则表达式 API响应解析: requests

数据分析与可视化: NLTK, Matplotlib, Seaborn, Pandas

Web应用程序: flask框架, 数据库, html, css等基本知识

预期效果
Project 目标

对于IT行业的热点领域, 结果是若干表格, 表格的分类有: 当前热度排序, 某领域的发展趋势, 领域的“替代品’’, “互补品”分析等:

对于IT从业者的最佳工作时间是利用Github数据的简单分析, 如同一repository的不同时间的commit, 同一时间的不同repository的commit等;
